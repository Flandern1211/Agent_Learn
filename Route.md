# Go AI Agent 开发与开源贡献规范

## 概述

本规范基于与用户的深入访谈制定，旨在为一名具有中等 Go 技能、对 AI Agent 开发感兴趣但缺乏相关开源贡献经验的开发者，提供一条清晰的学习和贡献路径。用户目标：学习使用 Go 进行 Agent 开发，并贡献于适合其能力范围、能提升认知的开源项目。

## 用户背景概要

- **Go 技能**：基础至中等水平，熟悉语法、接口、泛型、协程，使用过 net/http 和 Gin/Echo/Fiber
- **并发模型**：基础到中等掌握，能使用但未深入理解原理
- **数据库**：MySQL 基础 SQL，Redis 基础数据类型与异步队列
- **系统**：Linux 基础命令，Docker 基础（能看懂命令）
- **Python**：基础语法和高级特性（装饰器、异步）
- **Agent 开发知识**：了解 Skills、Function Call、MCP、Context 基本概念
- **MCP 经验**：用 Python 编写过简单 MCP 服务器，对协议大致了解
- **开源经验**：从未参与过
- **学习偏好**：循序渐进教程，每周可投入 20+ 小时
- **兴趣点**：记忆（Memory）、工具调用（Function Calling）、上下文管理、Skills 和 Subagent
- **项目偏好**：中型框架，代码质量高、社区活跃、技术栈匹配、有明确贡献指南

## 核心目标

1. **学习 Go AI Agent 开发**：掌握使用 Go 构建类似 Claude Code 的 AI Agent 所需的核心技术和框架。
2. **贡献开源项目**：在 3-6 个月内，向一个或多个 Go 相关开源项目提交有意义的贡献（代码或文档）。
3. **提升技术认知**：通过实践深入理解 AI Agent 架构、并发模型、协议设计和软件工程最佳实践。

## 技能差距分析

| 领域 | 当前水平 | 目标水平 | 关键差距 |
|------|----------|----------|----------|
| Go 并发高级特性 | 基础-中等 | 熟练 | goroutine 调度、channel 模式、竞争条件检测、性能优化 |
| Go 测试与文档 | 简单测试，经验有限 | 熟练 | 单元测试、集成测试、mock、基准测试、godoc 规范 |
| AI Agent 框架 | 不了解 Go 相关库 | 掌握至少一个主流框架 | langchaingo 或类似框架的使用和原理 |
| MCP 协议实现 | Python 简单实现，大致了解 | 能用 Go 实现完整 MCP 服务器 | SSE/JSON 传输、消息格式、错误处理、资源管理 |
| 开源协作流程 | 无经验 | 熟悉完整流程 | Git 工作流（fork、branch、PR、review）、issue 处理、CI/CD |
| 系统设计 | Web 服务经验 | 中型框架架构设计 | 模块划分、接口设计、可扩展性、可维护性 |

## 学习路径（分阶段）

### 第一阶段：巩固基础（2-3 周）
- **Go 并发深化**：阅读《Go 并发实战》相关章节，练习 channel 模式（fan-in/fan-out、worker pool）、context 用法、sync 包高级用法。
- **测试与文档**：编写带有完整测试的示例项目，学习 testify、gomock，实践 table-driven tests，生成 godoc。
- **MCP 协议深入学习**：阅读官方 MCP 规范，用 Go 实现一个简单的 MCP 服务器（提供当前时间、文件列表等工具）。

### 第二阶段：AI Agent 框架入门（3-4 周）
- **langchaingo 学习**：通过官方示例和教程，掌握其核心组件：LLM 调用、提示模板、链（chains）、工具（tools）、记忆（memory）。
- **构建简单 Agent**：使用 langchaingo 创建一个能调用外部 API（如天气、计算）的 CLI Agent。
- **理解现有项目**：选择 1-2 个使用 langchaingo 的中型开源项目，阅读其源码，理解架构。

### 第三阶段：贡献准备（2-3 周）
- **开源工具链熟悉**：练习 GitHub 协作流程（fork、clone、branch、commit、PR），学习 rebase、解决冲突。
- **目标项目筛选**：根据以下标准筛选 3-5 个潜在项目：
  - 使用 Go 语言，涉及 AI Agent 或相关基础设施
  - 有 `good-first-issue` 标签，社区活跃
  - 代码质量高，文档齐全
  - 规模适中（500-5000 行核心代码）
- **项目深度阅读**：选择其中一个项目，仔细阅读源码、文档、过往 issue 和 PR，理解其设计哲学和代码风格。

### 第四阶段：首次贡献（3-4 周）
- **从简单任务开始**：修复文档错别字、补充示例、改进测试覆盖率。
- **逐步提升**：处理小的 bug fix（需编写测试）、实现简单的功能增强。
- **参与社区**：在 issue 中提问、回答他人问题，了解项目维护者的期望。

### 第五阶段：深度贡献（持续）
- **承担功能开发**：在熟悉项目后，认领一个功能需求，独立设计并实现。
- **代码审查**：学习 review 他人的 PR，提出建设性意见。
- **成为常驻贡献者**：定期关注项目动态，持续贡献，争取成为 maintainer。

## 推荐项目清单

基于用户兴趣（AI Agent、中型框架）和能力范围，以下项目值得考虑：

### 主要目标
1. **langchaingo**（https://github.com/tmc/langchaingo）
   - Go 版的 LangChain，正在快速发展，社区活跃
   - 有许多 `good-first-issue`，涉及工具、记忆、链等组件
   - 代码质量较高，文档逐步完善
   - 与用户兴趣高度匹配

2. **mcp-go**（https://github.com/modelcontextprotocol/go-server）
   - 官方的 Go MCP 服务器 SDK
   - 用户已有 MCP 经验，可快速上手
   - 项目规模适中，适合贡献工具实现、示例或文档

3. **claude-code** 的 Go 相关组件
   - 探索 Claude Code 生态中是否有用 Go 编写的组件或插件
   - 例如：MCP 服务器、工具库等

### 备选项目
4. **griptape-go**（如果存在）
   - Go 版的 Griptape 框架
   - 可能需要调研是否有活跃项目

5. **go-openai**（https://github.com/sashabaranov/go-openai）
   - OpenAI API 的 Go SDK
   - 可作为工具调用基础，贡献示例或扩展功能

6. **其他 AI 基础设施**：向量数据库客户端（如 go-pgvector）、提示模板引擎等。

## 贡献策略

### 短期（1-2 个月）
- 在 langchaingo 中寻找与“记忆”或“工具调用”相关的 `good-first-issue`
- 提交 2-3 个小型 PR，熟悉项目流程
- 目标：被合并至少一个 PR，获得维护者信任

### 中期（3-4 个月）
- 认领一个中等复杂度的功能，如实现一种新的记忆存储后端（Redis 记忆）
- 编写完整测试和文档
- 参与代码审查，了解项目质量标准

### 长期（5-6 个月）
- 主导一个功能模块的设计与实现
- 协助维护 issue 列表，帮助新贡献者
- 考虑在熟悉后，向其他相关项目扩展贡献

## 潜在挑战与应对

| 挑战 | 应对策略 |
|------|----------|
| 开源协作流程不熟悉 | 先通过小型文档 PR 练习流程，观看相关教程视频 |
| 代码审查意见多 | 保持开放心态，将反馈视为学习机会，逐条回应 |
| 项目代码复杂难懂 | 从简单模块入手，多与维护者沟通，绘制模块关系图 |
| 时间管理 | 制定每周计划，固定投入时间，避免过度分散 |
| 技术深度不足 | 针对特定领域（如并发、协议）进行专项学习 |

## 成功衡量标准

- **学习成果**：能用 Go 独立实现一个具备工具调用、记忆和上下文管理功能的 AI Agent
- **贡献成果**：在目标项目中累计提交 ≥5 个合并的 PR，其中至少 1 个为功能实现
- **认知提升**：能清晰解释所选项目的架构设计、并发模型和扩展方式
- **社区参与**：在项目讨论区活跃，帮助解决至少 3 个其他用户的 issue
